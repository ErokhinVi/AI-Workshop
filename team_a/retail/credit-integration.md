# Памятка по стыковке — фича «Кредиты», команда A

Вкладка «Кредиты» в блоке retail готова. Сама она решение не принимает: за
решением ходит в cib, за хранением заявки — в backend. Чтобы фича заработала
целиком, нужны части от обоих коллег. Ниже — что retail отправляет и что ждёт
в ответ.

## Никите — блок cib

Сделать обработчик `POST /credit/decide`.

Получает от retail:

```
{ "client_id": "<id клиента>", "amount_rub": 150000, "term_months": 12 }
```

`term_months` — одно из: 6, 12, 24, 36.

Должен вернуть со статусом 200:

```
{ "decision": "approve" или "decline", "reason": "<человеческое объяснение>" }
```

- `decision` — строго `"approve"` или `"decline"`.
- Решение принимается по самому клиенту, не случайно. Данные клиента cib берёт
  у backend: `GET /clients/{client_id}` — эта ручка уже есть. Там доход
  (`income_rub`), остаток на счёте (`balance_rub`), история просрочек
  (`has_overdue_history`), сегмент (`segment`).
- `reason` — короткое человеческое объяснение. При отказе обязательно понятно,
  почему именно, а не сухое «отказано».
- retail ждёт ответ до 40 секунд — нужно уложиться, в идеале за пару секунд.

Ещё: завести кредитный продукт в каталоге cib (список `PRODUCTS`).

## Ивану — блок backend

Сделать две ручки.

`POST /credit-applications` — сохранить поданную заявку. retail присылает:

```
{ "client_id": "<id>", "amount_rub": 150000, "term_months": 12,
  "decision": "approve", "reason": "..." }
```

Ответ retail не проверяет — достаточно вернуть `{ "status": "ok" }`.

`GET /credit-applications` — вернуть список всех поданных заявок (по образцу
`GET /transactions/{client_id}`). Хранить в памяти, как клиентов и транзакции.

## Как всё сходится

- retail → cib `POST /credit/decide` — за решением
- retail → backend `POST /credit-applications` — за хранением заявки
- cib → backend `GET /clients/{id}` — за данными клиента (ручка уже есть)

retail уже вызывает обе ручки. Пока их нет — вкладка «Кредиты» не падает,
а показывает клиенту «банк ещё готовит решение». Как только cib и backend
сделают свои ручки, фича заработает у всей команды сама — без правок в retail.
