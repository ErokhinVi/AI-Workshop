# Инвест-рекомендации для премиум-клиентов

> Главный мульти-блочный кейс воркшопа. ~60-90 минут. Пройти его — значит увидеть как все шесть блоков правления работают вместе на одной задаче.
>
> Текущий стейт банка не описан здесь намеренно. Что уже есть в каталоге CIB, какие сегменты в seed-клиентах, что собрано после `case_01_credit` — посмотрите сами в своих блоках и через INBOX у соседей.

## Чего не хватает

Полноценного **инвест-сервиса** в мобильном банке для премиум-клиентов. Когда клиент класса premium заходит в приложение, он должен видеть подобранную **под него** рекомендацию — какие инструменты ему сейчас стоит рассмотреть, на основе его профиля.

После того как он «купил» инструмент, ему должен прийти **отчёт** — что он купил, сколько комиссия, какая ожидаемая доходность, в чём риски. Понятным человеческим языком, не финансовым жаргоном.

Правлению нужна **сводная картина** в реальном времени — сколько premium-клиентов в инвест-сервисе, какой объём активов под управлением, средняя комиссия.

## Что считаем готовым

1. **CIB**: каталог продуктов расширен инвест-инструментами разных уровней риска (минимум 5 продуктов: облигации, ETF, акции голубых фишек, что-то структурное, что-то фондовое). Ручка `/invest/recommend?client_id=...` возвращает 3-5 продуктов под клиента.
2. **Risk**: появилась ручка `/risk-appetite/{client_id}` — определяет risk-profile клиента (`conservative` / `balanced` / `aggressive`) на основе его профиля. **Важно:** для определения профиля используется LLM (через IT) — модель смотрит на клиента и объясняет почему такой профиль.
3. **Retail**: в мобильном UI для premium-клиентов появилась вкладка «Инвестиции» с рекомендациями. Можно «купить» инструмент — это создаёт запись в портфеле клиента.
4. **Finance**: ручка `/report/client/{id}/portfolio` возвращает читаемый HTML-отчёт по портфелю клиента: сводка человеческим языком (через LLM из IT), таблица активов, комиссии, доходность.
5. **CEO**: в `/dashboard` плитка «Премиум-инвест»: число клиентов с инвест-портфелем, объём AUM (assets under management), средняя комиссия.
6. **Live-табло воркшопа**: минимум 5 активных связей между блоками. Кооперация ≥ 60%.

## Как считаем что задача интересно прошла

Проверка — прогон одного полного сценария:
1. Берём из seed клиента Анну Морозову (id `c-00042`, premium, balance 8.4 млн).
2. Заходим в её мобильный кабинет (UI Retail).
3. Открываем вкладку «Инвестиции» — должна появиться рекомендация на 3-5 продуктов с объяснением «почему именно эти».
4. «Покупаем» один инструмент.
5. Получаем отчёт от Finance — на одну страницу, на русском, с понятной сводкой.
6. Возвращаемся в дашборд CEO — там плитка «Премиум-инвест» обновилась: счётчик клиентов вырос на 1, AUM выросло на сумму инвестиции.

Если каждый из этих шагов сработал — значит правление построило **полную цепочку** клиентского сценария и увидело свой банк как единое работающее целое.

## Подсказки

- Это шесть блоков, не один. Никто не тянет всё в одиночку. Хороший признак — каждый блок добавляет один-два эндпоинта и пишет соседям 2-3 запроса в INBOX.
- LLM полезен в трёх местах: определение risk-аппетита (Risk), краткое описание рекомендации (CIB), сводка отчёта (Finance). Везде где нужен «человеческий язык».
- Контракты на форматы данных (`/contracts/api_specs/*.yaml`) сильно сэкономят время на этапе интеграции. Договоритесь о формате карточки продукта **до того как** начнёте кодить.
- Если что-то не работает — на дашборде сразу видно где разрыв. Если стрелка `Retail → Risk` стала жёлтой и долго висит, значит Risk не отвечает на запрос.

## Когда готово

1. Полный прогон сценария с Анной Морозовой работает.
2. На воркшоп-дашборде минимум 5 активных связей и кооперация ≥ 60%.
3. Припишите в `dashboard/events.jsonl`:
   ```json
   {"block":"ceo","kind":"case_done","text":"case_02_invest готов — премиум-цепочка собрана","ts":<unixtime>}
   ```
4. Переходите к **глобальной цели дня** (`cases/goal.md`).
