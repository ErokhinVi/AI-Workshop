# Кейсы воркшопа

В этой папке лежат **бизнес-задачи** для правления — короткие постановки в свободной форме. Каждый кейс — это отдельная папка с одним `README.md` внутри, написанным как заказ от руководства.

Топ-менеджер открывает кейс, говорит своему AI-помощнику «возьмись за это», и дальше агент сам разбирается: к кому из соседних блоков пойти через INBOX, что согласовать в `/contracts/`, какой UI поправить, какую ручку поднять. Корневой `CLAUDE.md` уже учит агента, как работать с распределённой задачей.

## Структура

| Что | Что содержит |
|---|---|
| `_seed/` | стартовая среда — синтетические клиенты, транзакции, кредитная история и скрипт генерации |
| `case_01_credit/` | разогревочный кейс: добавить выдачу кредита в существующий UI |
| `case_02_invest/` | главный мульти-блочный кейс: процесс инвест-рекомендаций для премиум-клиентов |
| `goal.md` | глобальная цель дня (заготовка, финализируется ближе к воркшопу) |

## Как кейсы устроены внутри

В каждом `case_*/README.md` две вещи:

1. **Чего не хватает** — что нужно сделать.
2. **Что считаем готовым** — критерии успеха.

Никаких пошаговых playbook-ов. И **никакого описания текущего стейта банка** — узнавать, что уже есть, топы должны сами через свои блоки и через INBOX у соседей. В этом и смысл воркшопа.

## Стартовая среда

Перед началом дня поддержка прогоняет `python cases/_seed/load_into_blocks.py` — это заливает synthetic-клиентов в блоки. Что именно при этом подгружается и как используется — топы выясняют сами.
