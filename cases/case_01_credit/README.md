# Кредит в мобильном банке (разогрев)

> Это первый кейс воркшопа. Цель — за ~30 минут пройти весь жизненный цикл фичи и почувствовать как агенты блоков работают вместе.

## Что есть сейчас

В мобильном приложении банка (Retail, ручка `/`) клиенты могут делать переводы. Открывается базовая страница со списком последних транзакций и формой «От → Кому → Сумма → Перевести».

В банке загружено **500 клиентов** с разной историей и разным доходом — данные в seed (см. `cases/_seed/`). Часть из них уже имеет кредитную историю — есть и аккуратные платильщики, и проблемные.

Доступные API на старте дня:
- `Retail` — `/clients`, `/clients/{id}`, `/transactions/{id}`, `/api/transfer`.
- `CIB` — `/products` (каталог банковских продуктов).
- `Risk` — `/score/{client_id}` (вернёт `risk_score` от 0 до 1).
- `Finance` — `/balance/{client_id}`, `/limits` (кредитные лимиты по сегментам).
- `IT` — `/llm/ask` (прокси к внешнему LLM).
- `CEO` — `/dashboard` (агрегатор для главного экрана).

## Чего не хватает

В UI нет **возможности подать заявку на кредит**. Это критично — мы видим в seed-данных клиентов с хорошим доходом и без активных кредитов, и ничего им не предлагаем.

## Что считаем готовым

1. В мобильном UI Retail появилась рабочая вкладка «Кредит» рядом с «Переводы».
2. Клиент вводит сумму и срок, нажимает «Подать заявку», получает решение **в течение 5 секунд** (одобрено/отказ + ставка/сумма + причина).
3. Решение основано на реальных данных клиента из seed: доход, баланс, история. Не на random.
4. Если отказ — клиент видит **понятное человеку объяснение** на одну фразу, не код-ошибки. (Подсказка: для генерации объяснения может пригодиться LLM через IT.)
5. На дашборде CEO в плитке `/dashboard` появилась цифра «Заявок на кредит сегодня» и «Процент одобрения».
6. Live-табло воркшопа показывает минимум 3 связи между блоками (видно по зелёным/жёлтым стрелкам).

## Как считаем что задача интересно прошла

После того как UI работает, выберите вместе с владельцем UI трёх клиентов из seed:
- Одного **очевидно платежеспособного** (premium-сегмент, доход выше медианы).
- Одного **очевидно проблемного** (есть просрочки в `credit_history`).
- Одного **на грани** (mass-сегмент, средний доход).

Прогоните каждого через UI. Все три решения должны быть осмысленными и объяснимыми. Если по проблемному клиенту банк выдаёт кредит без оговорок — что-то не так в риск-модели, нужно идти к Роланду.

## Технические границы

- Любые внешние API — через IT блок (`NEIGHBOR_IT/llm/ask`). Не дёргайте OpenAI напрямую из своего блока.
- Чужие папки read-only. Если нужна ручка у соседа — пишите в его INBOX через стандартный канал.
- Если ручки соседа возвращают 502 (он ещё не задеплоил) — не делайте мок «как будто бы было». Лучше остановиться и попросить, либо явно вернуть «соседский сервис не готов».

## Когда готово

1. Покажите владельцу UI-приложения (Иван), дайте попробовать на двух клиентах из seed.
2. Если всё работает — припишите одну строку в `dashboard/events.jsonl`:
   ```json
   {"block":"<ваш блок>","kind":"case_done","text":"case_01_credit готов","ts":<unixtime>}
   ```
   На главном экране воркшопа загорится зелёный пульс.
3. Переходите к `case_02_invest`.
