# .claude/templates/

Здесь лежат заготовки `settings.local.json` для каждого блока.

## Зачем

Это **жёсткая защита** — Claude Code физически не сможет редактировать
файлы вне твоей папки, даже если ты его попросишь. Ошибка будет
возвращена встроенным механизмом permissions, не моей инструкцией.

## Когда применять

**Сам Claude Code применяет автоматически** при онбординге — после того
как ты представился, агент скопирует нужный template в
`.claude/settings.local.json`.

Если по какой-то причине это не сработало, можно вручную:

```bash
# для блока CIB
cp .claude/templates/settings-cib.json .claude/settings.local.json
```

После этого перезапусти Claude Code — он подхватит новые permissions.

## Что внутри

Каждый файл блокирует:
- редактирование папок других блоков (`Edit(другой/**)`, `Write(другой/**)`);
- редактирование `contracts/shared.py` и чужих OpenAPI-спек;
- редактирование `infra/`, `docker-compose.yml`, root-md файлов.

И разрешает:
- редактирование своей папки;
- редактирование своего OpenAPI;
- запись в INBOX (это общественная зона);
- чтение всего;
- запуск стандартных команд (docker-compose, git, python и т.д.).
