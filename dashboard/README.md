# Live Dashboard — AI-воркшоп правления Райффайзен

> Один экран на проектор. Показывает в реальном времени, как 6 блоков
> правления взаимодействуют через INBOX, как растёт код и метрики банка.

## Что видно

- **Граф**: 6 кругов (один на блок), размер растёт от объёма работы.
  Линии — переписка через INBOX, толщина = количество сообщений,
  цвет = статус (жёлтый = открыто, синий = ответили, зелёный = договорились,
  красный = отказ).
- **Метрики банка** (правая панель): строки кода, коммиты, открытые
  запросы, договорённости, агрегированный **Bank score**.
- **Лента событий**: последние коммиты, INBOX-сообщения и (опциональные)
  бизнес-события агентов.

Дашборд **сам** подтягивает изменения через `git fetch` каждые 5 секунд
и пересканирует репо. Никто из топов и их агентов не должен ничего
явно «отправлять в дашборд» — он наблюдает за репозиторием.

## Запуск (локально, на ноутбуке поддержки)

```bash
cd raif-bank-sandbox
pip install fastapi 'uvicorn[standard]'
python dashboard/server.py
```

Открой [http://localhost:9000](http://localhost:9000). Порт **9000** — вне
диапазона блоков (8000–8059), не пересекается ни с кем.

Опции:

```bash
python dashboard/server.py --port 9000 --no-fetch    # не дёргать GitHub
python dashboard/server.py --repo /path/to/clone     # клон в другом месте
```

## Запуск в Docker

```bash
docker build -t raif-dashboard -f dashboard/Dockerfile .
docker run --rm -p 9000:9000 -v "$PWD":/repo raif-dashboard
```

## Опциональные бизнес-события (для агентов)

Если агент хочет, чтобы важное действие пользователя появилось в ленте
своим осмысленным текстом — он может дописать одну строчку в
`dashboard/events.jsonl`:

```jsonl
{"block":"cib","kind":"build","text":"Никита запустил scoring API","ts":1714750000}
```

Это **не обязательно**. Дашборд и без этого видит коммиты и INBOX.

## Авто-деплой на GitHub Pages

В репо лежит `.github/workflows/dashboard.yml` — он на каждый push в
`main` пересобирает `state.json` и деплоит свежую страничку. Никому
ничего нажимать не нужно: топ закоммитил → через ~30–60 секунд экран
на проекторе показывает новую картинку.

### Однократная настройка (5 секунд в UI GitHub)

1. Открой [Settings → Pages](https://github.com/ErokhinVi/AI-Workshop/settings/pages).
2. **Source: GitHub Actions**.
3. Сохрани.

После первого успешного запуска workflow URL будет:
`https://ErokhinVi.github.io/AI-Workshop/`

Это и есть «постоянная страница» — открываешь её на ноуте поддержки в
полноэкранном браузере и проецируешь.

### Что триггерит ребилд

- Любой push в `main` (любой коммит любого члена правления).
- Cron раз в 5 минут — страховка, если push-событие почему-то пропустили.
- Ручной запуск через `Actions → Dashboard → Run workflow`.

### Как frontend получает данные

Фетчит `state.json` относительным путём с cache-busting (`?t=...`).
Это работает одинаково и через GitHub Pages (где `state.json` —
артефакт сборки), и через локальный `server.py` (там есть тот же роут).

## Что НЕ делает дашборд

- Не пишет в репо (read-only).
- Не вмешивается в работу агентов.
- Не требует авторизации (это локальная демо-сцена).
