## Деплой — автоматический

Твой блок задеплоен на Render: **https://raif-cib.onrender.com**.

После каждого `git push` в main GitHub Action `Deploy services via
Render Deploy Hooks` пересобирает только тот блок, чья папка тронута
(твой код в `/cib/` → пересборка только `raif-cib`). Сборка
~2-4 минуты. Подробности — `/DEPLOY.md` в корне репо.

### Что делать после push

```bash
git push origin main
python dashboard/wait_deploy.py raif-cib
```

Helper пингует /health пока сервис не ответит 200. Когда вернёт exit 0
— скажи пользователю человеческим языком:

> «Готово, [имя]. Твой блок обновился. Открой:
>  • https://raif-cib.onrender.com — главная
>  • https://raif-cib.onrender.com/docs — Swagger со всеми ручками»

Опционально, для эффекта на live-табло у Виталия:

```bash

```

## Как общаться с соседями

Render инжектит URL-ы соседей в env. Используй так:

```python
import os, httpx
risk_url = os.environ.get("NEIGHBOR_RISK", "http://localhost:8050")
async with httpx.AsyncClient(timeout=5.0) as client:
    r = await client.get(f"{risk_url}/score/c-01000")
    return r.json()
```

Локально (docker-compose) переменных NEIGHBOR_* нет — fallback на
`http://localhost:<порт_соседа>` (см. порты в `TEAM.md`). На Render —
на публичные URL соседей.

LLM (внешний OpenAI) — ходи только через IT блок:
`httpx.post(NEIGHBOR_IT + '/llm/ask', json={'prompt': '...', 'requester': 'cib'})`.
Никакого прямого `OPENAI_API_KEY` у тебя нет, ключ у IT.

### Если сосед ответил 502/503

Не моки, не делай fallback с фейковыми данными. Скажи топу:

> «У [имя соседа] эта ручка ещё не готова. Хочешь, напишу ему в INBOX
>  с просьбой поднять?»

И если да — пиши через `INBOX/to_<сосед>.md`. Это и есть смысл воркшопа —
прочувствовать что блоки реально друг от друга зависят.
